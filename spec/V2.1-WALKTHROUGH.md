# Implementación Parcial v2.1 (Fase 6)

He avanzado con la implementación de dos de las iniciativas de más rápido retorno y nula dependencia de terceros que acordamos para la `v2.1`, cubriendo los *Gaps* del proyecto original.

## 1. Exportación de CSV Administrativa (G-07)
Añadida la capacidad para que el personal genere reportes operacionales.
- **Botones Dinámicos:** Se agregó un botón *Exportar CSV* en las vistas de **Órdenes**, **Clientes** y **Productos**.
- **Endpoint Optimizado:** En vez de delegar el trabajo a una librería frontend que colapse la memoria del navegador celular del Administrador, creé un backend endpoint unificado en `/api/admin/export/[type]`, protegido por el RBAC (`requireAdminSession`). Esto extrae los datos usando Prisma Streams y los devuelve con cabecera `text/csv`.

## 2. Promedios de Rating Desnormalizados (G-10)
Se eliminó el cálculo en memoria de reseñas para hacer más rápido el escaneo del catálogo:
- **Cambios en Base de Datos:** Se agregaron los campos `averageRating` (Float) y `totalReviews` (Int) a la tabla madre `Product`.
- **Nuevo Action Cliente:** Reemplacé el código simulado en `ReviewForm` con el Server Action real `addReviewAction`, logrando una inserción veraz en base de datos bajo un estado `PENDING`.
- **Efecto Cascada:** Se reformateó `moderateReviewAction`. Ahora, cada vez que un Administrador apruebe o rechace una reseña, de fondo se hace una agregación que re-calcula matemáticamente el `averageRating` del Producto y lo _escribe_ en base de datos. De estar forma, consultar 10,000 productos no forzará a la base a agrupar e iterar por 100,000 reseñas.
